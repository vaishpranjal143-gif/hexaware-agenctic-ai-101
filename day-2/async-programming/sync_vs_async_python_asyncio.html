Python · asyncio · concurrency

Sync vs Async

A visual guide to how blocking and non-blocking Python code executes differently — powered by asyncio and the event loop.

01
The analogy

Synchronous · blocking

One thing at a time

Like a single-file queue — each person is fully served before the next one starts. The thread freezes while waiting for I/O (disk, network, DB). Simple to reason about, but wastes CPU cycles on waiting.

Asynchronous · non-blocking

Juggling multiple tasks

Like a chef managing many dishes — when one is in the oven, you prep another. The event loop yields control at each await, switching to other tasks while I/O completes.

02
Code side by side
sync_example.py
import time
def fetch_data(name, delay):
print(f"Starting {name}...")
time.sleep(delay) # ← blocks thread
print(f"{name} done!")
def main():
fetch_data("Task A", 3)
fetch_data("Task B", 2)
fetch_data("Task C", 1)
# sequential: 3+2+1 = 6s total
main()

async_example.py
import asyncio
async def fetch_data(name, delay):
print(f"Starting {name}...")
await asyncio.sleep(delay) # ← yields
print(f"{name} done!")
async def main():
await asyncio.gather(
fetch_data("Task A", 3),
fetch_data("Task B", 2),
fetch_data("Task C", 1),
)
# concurrent: max(3,2,1) = 3s total
asyncio.run(main())

$ python async_example.py

Starting Task B...
Starting Task C...
✓ Task C done! at t=1s
✓ Task B done! at t=2s
✓ Task A done! at t=3s
